给王家林-讲师-讲课建议-01：（学术是严谨的，需要不断探究更好解决方案。－－－－－－因为有你，我们每天都在长大）

1.上课前先严格备好整套课件，包括ppt,源码，概念名词，专业述语，一定要有专门的解释说明，以文字的形式在屏幕上显示出来，有些最好配带图文说明
2.学术是严谨的，做为技术传播者，千万不要有这样的词出现，随便试下、只是输错了、原来是那样的、等说辞，有儿戏，不专业的感觉
3.同一个意思，如果表述的顺序，语速，语调，语气不同，传达给受众的效果是完全不一样的，人的思维也常常会短路，所以一定要备课充足，不能临时想到啥讲啥，临时写代码，往往会乱七八遭或花很多时间，
  而且很有可能讲不清楚，道不明白，这就是需要教科书的目的
4.图文说明写出来，对着讲 最好是分段说明，每一步是干什么的，有明显的层次结构，归纳总结（试想，一个刚开始学的学员，听你操做两个小时的代码，哪还记得哪跟哪，如果刚靠看一遍，那忘了至少40%以上，对受众效果大打折扣）
  一、本课时学哪些内容，这些内容是什么东西，用来干什么的，名词，概念解释，关联关系
  二、配合实际案例，代码分析，代码一定要有注释，如java之所以在全世界推广一个很重要的原因，jdk源码，jdk文案说的很明白，java 类上注释就把该类的功能说的很详细
  	  举例，配上关键注释就更好理解了：
  	  	import scala.util.control.Breaks._

			object _breakable {
			  
			  def main(args: Array[String]): Unit = {
			   //beak体，用来跳转
			    breakable {
			     for(i <- 0 to 10){
			       if(i ==4) break   //跳出break块
			       println(i)
			     }

			     println("for end");//beak后，不会执行
			    }

			     println("beak end");//beak后还会执行
			  }
			}
  三、继续看，以后提更多感受，讲的不对的地方，希望讲师海涵，忠心的祝愿讲师和学员，都能大展鸿图
